# Web Development Checklist

## When to use this skill
- Before "shipping" a new page or section.
- To ensure all semantic tags are used correctly.
- When performing a quality review (QA) of a project.

## Workflow
1.  **Tag Review**: Verify that `<main>`, `<nav>`, `<article>`, and `<footer>` are used as per the guidebook.
2.  **Attribute Audit**: Ensure all `<img>` have `alt` tags, and interactive elements have appropriate states.
3.  **SEO Check**: Verify headings (`<h1>` to `<h6>`) follow a logical hierarchy and that meta tags are correctly set.
4.  **Accessibility**: Ensure buttons and links are descriptive.

## Guidebook Sections
- **HTML Basics**: Standard structure.
- **Structural/Semantic Elements**: Modern layout tags.
- **Form Handling**: Inputs, labels, were they implemented correctly?
- **Formatting**: Styling text for readability and impact.

## Instructions
- Check against the `Learn-Web-Development-Checklist` resource during every final review.
- Prioritize **Semantic HTML** (it's the foundation for SEO and accessibility).
- Never skip the "QA" phase of a project.
