# Senior App Frontend Guidelines

## When to use this sub-skill
- When transitioning from a static web page to a complex application frontend (e.g., connecting a database, state management).
- When writing components that need to scale.

## Workflow
1.  **Architecture**: Enforce a strict separation of concerns (Logic vs UI).
2.  **State Management**: Use performant patterns to avoid unnecessary re-renders in the app.
3.  **Clean Code**: Maintain DRY (Don't Repeat Yourself) code with modular, reusable components.

## Instructions
- Treat web projects as scalable software.
- Use strict typing (TypeScript principles) implicitly to ensure robustness.
