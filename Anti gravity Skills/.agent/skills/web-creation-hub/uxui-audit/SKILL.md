# UX/UI Audit Principles

## When to use this sub-skill
- Before finalizing a web/app design.
- To detect UX antipatterns that confuse users.
- When transforming a good design into an "Apple-level" premium experience.

## Workflow
1.  **Visual Inspection**: Ensure alignment, consistent spacing, and visual hierarchy.
2.  **Interaction Check**: Ensure all buttons appear clickable, forms are intuitive, and there are no dead-ends.
3.  **Pattern Verification**: Verify that the design adheres to established scientific UX principles (e.g., Fitts's Law, Hick's Law).

## Instructions
- Acting as a ruthless UX auditor, point out any design decision that causes cognitive load.
- Never settle for "it just looks good"—it must also be deeply usable.
